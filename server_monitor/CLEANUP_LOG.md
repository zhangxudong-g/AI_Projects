# 数据库清理说明

## 问题
数据库中存储了5102条带有错误时间戳的记录（时间戳显示为1970年），导致：
- GPU历史峰值API查询无法找到有效数据
- 所有时间范围查询都无法匹配到记录

## 解决方案
1. 修复了monitor.py中的时间戳生成逻辑（PR #xx）
2. 清理了数据库中所有错误时间戳的记录
3. 现在系统将使用正确的时间戳格式存储新数据

## 结果
- 已删除5102条错误记录
- 数据库现在是干净的
- 新的监控数据将使用正确的时间戳
- API查询将能够正常返回数据
