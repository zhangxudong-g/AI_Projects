# 配置说明

## 环境变量配置

系统支持通过环境变量进行配置。您可以复制 `.env.example` 文件为 `.env` 并根据需要修改配置项。

### 可用配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DELETE_CASE_FILES_SYNC` | `false` | 删除案例时是否同步删除文件 |

### DELETE_CASE_FILES_SYNC 详细说明

- **`false`** (默认): 删除案例时仅删除数据库记录，保留关联的文件（源代码、Wiki 文档、配置文件）
- **`true`**: 删除案例时同时删除数据库记录和所有关联文件

**使用场景：**
- 开发/测试环境：建议设置为 `false`，便于调试和恢复数据
- 生产环境：建议设置为 `true`，避免磁盘空间浪费

**示例：**
```bash
# .env 文件
DELETE_CASE_FILES_SYNC=true
```

**注意：** 
- 文件删除操作会记录日志
- 如果文件删除失败（如权限问题），数据库记录仍会被删除
- 文件路径必须在 `data/cases/` 目录下才会被删除（安全保护）
