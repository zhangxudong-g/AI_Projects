# 芯片数据手册 → 业务层接口生成

读取芯片数据手册和现有 vendor 驱动代码，生成业务层接口模块（.h/.c）和接口规格文档。生成的接口封装 vendor 层的寄存器操作，为业务逻辑生成（gen-project skill）提供干净的调用表面。

## 架构背景

```
vendor/（厂家驱动，寄存器操作）→ 接口层（本次生成）→ 业务逻辑（main.c，gen-project 生成）
```

- **vendor/** — 芯片厂家提供的寄存器级驱动（只读，不修改）
- **接口层** — 封装 vendor 操作，暴露业务能力（本次生成）
- **业务逻辑** — 只调接口，不直接操作寄存器

> **完整范例**：生成接口前，先读取 `references/example-interface.md`，对照其中的 vendor→接口转换范例理解输出格式。范例展示了头文件注释、实现封装、规格文档的完整写法，以及 vendor 层和接口层的关键差异对照表。

## 执行协议

### Step 1: 确定输入源

询问用户或从 CLAUDE.md / 项目文档中识别以下信息：

1. **数据手册位置** — `docs/datasheets/` 下的 markdown 文件（芯片数据手册、MCU 说明）
2. **vendor 驱动位置** — 项目中已有的芯片驱动代码目录
3. **接口模块划分** — 需要生成哪些接口模块（如 power/led/key/display/sensor 等）
4. **MCU 和编译器约束** — 架构（8051/ARM/RISC-V）、ROM/RAM 大小、编译器（C51/Keil/GCC）
5. **输出路径** — 接口文件存放位置

如果没有明确指定，使用项目约定：
- 数据手册：`docs/datasheets/`
- vendor 驱动：`vendor/` 目录
- 输出路径：与 vendor 同级目录
- 规格文档：`docs/interfaces/`

### Step 2: 读取数据手册

读取所有相关数据手册，提取关键信息：

- 芯片 IIC/SPI 地址格式（7 位 / 10 位）
- 寄存器地址映射表（地址、名称、位域、读写属性）
- 功能参数（电压/电流/功率预设值、对应的寄存器值、换算公式）
- 保护机制（过温、过流、欠压、短路）相关寄存器
- 页面切换 / 存储区切换机制
- 初始化序列和上电时序
- 状态寄存器位域定义
- 电气特性（延时要求、阈值等）

### Step 3: 读取现有 vendor 代码

读取 vendor 驱动代码，理解实现模式：

- 通信函数签名（IIC/SPI 读写函数）
- 页面切换方式（如有）
- GPIO 操作模式（sbit / 宏 / 函数）
- 全局变量和配置结构
- 现有的外设驱动（LED、按键、显示等）
- 中断处理方式

### Step 4: 读取项目规范

读取以下文档（如存在）：
- `docs/guides/ai-dev-workflow.md` — 两阶段工作流定义
- `docs/guides/interface-separation.md` — 接口分离设计
- `CLAUDE.md` — 项目约定（引脚定义、编码规范等）

### Step 5: 设计接口模块

参考 `references/example-interface.md` 中的范例，对照 vendor 层的函数设计业务接口。

为 Step 1 确定的每个模块设计接口。设计原则：

1. **命名规则** — 模块名作为前缀，如 `{module}_init()`、`{module}_enable()`
2. **封装 vendor** — 所有底层操作通过 vendor 函数完成，接口层不做寄存器操作
3. **参数校验** — 在接口层完成，返回错误码
4. **返回值统一** — 0=成功，负数=错误码（-1=参数无效，-2=通信失败，等）
5. **阻塞标注** — 每个函数明确标注是否阻塞、是否 ISR 安全
6. **最小化接口** — 只暴露业务需要的函数，不暴露调试/测试接口

### Step 6: 生成接口头文件和实现

为每个模块生成 `.h` 和 `.c` 文件。参照范例中 `voltage.h` / `voltage.c` 的结构和风格。

**头文件注释格式**（每个公共函数必须遵循，范例中有完整示例）：

```c
/**
 * 函数简述
 * @param name  描述 (单位, 有效范围)
 * @return 0=成功, -1=参数无效, -2=通信失败
 * @note  阻塞/非阻塞, 耗时约Xms (原因)
 * @warning 不可在 ISR 中调用 / 可在 ISR 中调用
 * @pre   前置条件
 */
```

**实现文件原则**（范例中 `voltage.c` 展示了完整模式）：
- 内部调用 vendor 层通信函数
- 页面切换/存储区切换在实现内部处理
- 寄存器地址常量只在 `.c` 文件中定义，不出现在 `.h` 中
- vendor 层全局变量只在 `.c` 中引用

### Step 7: 生成接口规格文档

为每个模块生成规格文档到 `docs/interfaces/{module}-spec.md`。参照范例中 `voltage-spec.md` 的结构和风格。

每个函数一个 spec table：

```markdown
### function_name

| 属性     | 值                                        |
|----------|-------------------------------------------|
| 签名     | int function_name(type param)             |
| 参数     | param: 描述, 有效范围, 单位               |
| 返回值   | 0=成功, -1=参数无效, -2=通信失败          |
| ISR安全  | 是/否                                     |
| 阻塞     | 是/否, 耗时约 Xms (原因)                  |
| 前置条件 | xxx_init() 已调用                         |
| 副作用   | 修改的全局状态/寄存器                      |
| 测试     | 验证方法                                  |
| 备注     | 特殊说明（预设值映射、换算公式等）         |

#### vendor 调用路径
function_name() → vendor_func_1() → vendor_func_2()
```

### Step 8: 自检验证

生成完成后，逐项检查并输出报告：

**通信验证**：
- [ ] 每个芯片使用正确的 IIC/SPI 通信函数
- [ ] 地址格式处理正确（7 位 / 10 位）
- [ ] 页面切换/存储区切换使用正确（如适用）

**寄存器操作**：
- [ ] 寄存器地址与数据手册一致
- [ ] 写入后延时满足数据手册要求
- [ ] GPIO 引脚分配与原理图/MCU 定义一致

**接口隔离**（参照范例末尾的对照表）：
- [ ] `.h` 文件中没有 vendor 层全局变量（如 `IIC_ADDRESS`、`IIC_DAT`）
- [ ] `.h` 文件中没有寄存器地址常量（如 0x02、0x05）
- [ ] `.h` 文件没有 include vendor 层头文件（只 `.c` 中 include）
- [ ] 头文件只暴露函数声明和业务常量（如 VOLTAGE_5V，而非 0x4B）

**平台兼容**：
- [ ] 无动态内存分配（malloc/free）
- [ ] 无 float 运算（改为定点）
- [ ] 数据类型符合平台约束（unsigned char/unsigned int 等）
- [ ] 每个公共函数有完整注释

## 约束

1. **不修改 vendor 层** — vendor 代码为只读
2. **不修改业务逻辑** — main.c/int.c 属于 gen-project 范围
3. **平台兼容** — 符合目标 MCU 和编译器约束
4. **UTF-8 编码** — 所有新文件使用 UTF-8，注释使用中文
