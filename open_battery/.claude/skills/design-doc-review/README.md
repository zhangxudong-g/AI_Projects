# Design Document Review Skill

## 简介

这个 skill 专门用于评审嵌入式固件项目的设计文档，提供系统化、多维度的分析。

## 适用场景

### 1. AI 辅助开发工作流评审

**触发方式**：
```
review 这篇 AI 辅助开发工作流文档，从可行性、完整性、合理性等方面评估
```

**评审重点**：
- AI 能力边界是否明确
- 人工验证点是否设置
- 工作流程是否可执行
- 工具支持是否充分

### 2. 芯片驱动设计评审

**触发方式**：
```
检查这个 SW6301 驱动设计文档有什么问题
```

**评审重点**：
- 寄存器定义是否完整
- 时序约束是否明确
- 错误处理是否完善
- 接口设计是否合理

### 3. 需求规范评审

**触发方式**：
```
审查这个 YAML 需求规范，看看有没有遗漏
```

**评审重点**：
- 触发条件是否明确
- 边界条件是否定义
- 参数范围是否完整
- 优先级是否合理

### 4. 状态机设计评审

**触发方式**：
```
验证这个状态机设计，检查是否有死锁
```

**评审重点**：
- 初始状态是否定义
- 所有状态是否可达
- 是否存在死锁
- 转换条件是否完整

## 评审维度

本 skill 从以下 6 个维度进行评审：

| 维度 | 说明 | 关键检查项 |
|------|------|-----------|
| **可行性** | 技术上能否实现 | AI 能力边界、资源约束、硬件限制 |
| **完整性** | 是否覆盖所有必要内容 | 用例覆盖、错误处理、边界条件 |
| **合理性** | 设计是否合理 | 状态机逻辑、时序参数、资源分配 |
| **一致性** | 术语和引用是否统一 | 参数定义、接口规范、文档引用 |
| **实用性** | 是否易于理解和执行 | 示例、检查清单、工具支持 |
| **安全性** | 故障和保护机制 | FMEA、恢复策略、中断安全 |

## 输出示例

评审报告包含以下部分：

```markdown
## 📋 评审摘要

| 评审维度 | 评分 | 关键发现 |
|---------|------|---------|
| 可行性 | ⭐⭐⭐⭐☆ | AI 能力边界不够明确 |
| 完整性 | ⭐⭐⭐☆☆ | 缺少版本管理策略 |
| 合理性 | ⭐⭐⭐⭐☆ | 状态机有小缺陷 |
| 一致性 | ⭐⭐⭐⭐⭐ | 术语使用统一 |
| 实用性 | ⭐⭐⭐☆☆ | 缺少 Prompt 模板 |
| 安全性 | ⭐⭐⭐⭐☆ | 错误恢复策略待完善 |

## ✅ 优点与亮点
1. 结构清晰，三阶段划分合理
2. 提供了实用的模板和示例
...

## ⚠️ 关键问题

### P0 - 必须修复
1. **状态机缺少 FAULT 恢复路径**
   - 影响：故障后系统无法自动恢复
   - 建议：增加 FAULT → IDLE/CHARGING 转换

### P1 - 强烈建议
2. **缺少人工验证节点**
   - 影响：过度依赖 AI 可能导致质量问题
   - 建议：在阶段 1 和阶段 2 后增加人工审查

## 💡 改进建议

### 建议 1：增加需求细化阶段
**当前状态**：要求用户直接编写 YAML
**问题**：用户可能不熟悉 YAML 格式
**改进方案**：AI 通过对话引导用户补充细节
**示例**：...
```

## 嵌入式特定检查清单

本 skill 特别关注嵌入式项目的特殊性：

### 硬件约束
- [x] GPIO 资源分配是否冲突
- [x] 中断优先级是否合理
- [x] 时序参数是否满足数据手册
- [x] IIC/SPI 地址格式是否正确
- [x] 电气特性是否明确

### 软件质量
- [x] 状态机是否无死锁
- [x] ISR 中是否无阻塞调用
- [x] 内存使用是否在范围内
- [x] 延时函数是否平台适配
- [x] 边界条件是否明确

### 错误处理
- [x] 通信操作是否有超时
- [x] 重试策略是否明确
- [x] 错误码是否完整
- [x] 恢复策略是否可执行

## 使用技巧

### 1. 明确评审重点

如果你只关心特定方面，可以这样说：
```
重点评审状态机设计的合理性，看看有没有死锁
```

### 2. 对比式评审

如果有多个版本，可以这样说：
```
对比 v1.0 和 v2.0，看看改进了哪些问题
```

### 3. 增量评审

如果只改了部分内容，可以这样说：
```
我修改了状态机部分，review 一下改动是否合理
```

## 项目集成

这个 skill 已经集成到当前项目的 `.claude/skills/` 目录中。

Claude 会在以下情况自动触发此 skill：
- 用户提到 "review"、"评审"、"审查"、"检查" + 文档相关词汇
- 用户提供 `.md` / `.yaml` 文件并要求反馈
- 用户提到设计文档、需求规范、技术方案
