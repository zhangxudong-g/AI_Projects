# 充电宝（移动电源）开发技术分析

## 一、充电宝工作原理

### 核心组件
| 组件 | 说明 |
|------|------|
| 电芯 | 锂电池(Li-ion)或锂聚合物(Li-Po)，储存电能 |
| 升压电路 | Boost Converter，将电芯电压升压到USB标准5V（磷酸铁锂约3.2V） |
| 充电管理电路 | 控制输入充电电流，保护电池 |
| 保护电路 | 过充、过放、过流、短路保护 |

### 充电/放电过程
- **充电**：接通电源 → 充电管理芯片控制电流进入电芯 → 电能转化为化学能储存
- **放电**：连接设备 → 升压电路将电芯电压（磷酸铁锂约3.2V）升压至5V → 通过USB接口输出

### 关键指标
- **容量**（mAh）：电芯储存能力
- **转换率**：约80-90%，受电路效率影响
- **输出**（5V/X A）：决定充电速度

## 二、硬件组成与软件部分

| 模块 | 说明 | 是否需要写代码 |
|------|------|---------------|
| 电芯 | LiFePO4等 | ❌ 物理部件 |
| 保护电路(BMS) | 过充/过放/过流/短路保护 | ⚠️ 部分需要固件 |
| USB-C PD控制器 | 双向快充协议芯片 | ✅ 需要固件配置 |
| USB-A QC控制器 | 快充协议芯片 | ✅ 需要固件配置 |
| 无线充电模块(Qi2) | 15W发射端控制 | ✅ 通常有独立MCU |
| 升压/降压电路 | DC-DC转换 | ❌ 纯模拟/IC |
| 外壳/结构件 | 物理结构 | ❌ 无 |
| 显示/指示灯 | 电量LED或屏幕 | ✅ 驱动代码 |

## 三、主控程序

主控程序（固件/Firmware）是写在硬件内部MCU中的核心软件，负责整体协调和控制。

### 主控程序的核心职责

| 功能 | 说明 |
|------|------|
| 系统初始化 | 上电后初始化各外设模块 |
| 状态管理 | 记录电芯SOC、充放电状态 |
| 协议调度 | 与PD/Qi2芯片交互，协商充放电功率 |
| 端口检测 | 监测USB插入/拔出，切换输出模式 |
| 保护触发 | 异常时关断电路，触发BMS |
| 通信接口 | UART/I2C/SPI与各芯片通信 |

### 主控读取电芯状态的方式

```
电芯 ◄──物理导线──► 采样电路 ◄──ADC/I2C──► 主控MCU
```

| 监测内容 | 实现方式 |
|---------|---------|
| 电压 | 分压电阻 → ADC采样 |
| 电流 | 电流检测电阻(采样电阻) → ADC或专用IC |
| 温度 | NTC热敏电阻 → ADC |
| SOC(电量) | 通过电压/电流积分算法计算 |

> 主控程序通过读取BMS报告的数据来感知电芯状态，而不是直接"对接"电芯。

## 四、参考产品：ポポフmini（Swiftechie）

### 产品规格
| 项目 | 规格 |
|------|------|
| 电池类型 | 磷酸铁锂电池（LiFePO4） |
| 容量 | 80Wh / 25,000mAh |
| 尺寸 | 87×87×156mm |
| 重量 | 800g |
| 总输出 | 最大100W |

### 输出端口
- 无线充电：15W（Qi2）
- USB-A：最大18W
- USB-C ①：最大45W
- USB-C ②：最大65W（支持输入）

### 产品亮点
- 超长寿命：约2,000次循环
- 高安全性：LiFePO4电池，热失控风险低
- 免布线：无需施工，哪里都能用
- 适合ABW办公场景

## 五、固件框架（通用方案）

```
app/
├── main.c                 # 入口，初始化+主循环
├── config/
│   ├── board_config.h     # 硬件配置（引脚、时钟）
│   └── protocol_config.h  # PD/QC协议参数配置
├── driver/
│   ├── gpio.c/h           # GPIO驱动
│   ├── adc.c/h            # 电池电压/电流采样
│   ├── pwm.c/h            # 升压/降压控制
│   ├── timer.c/h          # 定时器（延时/PWM）
│   └── usb.c/h            # USB物理层驱动
├── protocol/
│   ├── pd_handler.c/h     # PD协议状态机
│   ├── qc_handler.c/h     # QC协议处理
│   └── fcp/scp.c/h        # 其他快充协议
├── bms/
│   ├── bms_core.c/h       # 电池管理核心（SOC计算）
│   ├── protection.c/h     # 保护逻辑（过充/过放/过流/短路）
│   └── cell_balance.c/h  # 电芯均衡（多节电芯时）
├── power/
│   ├── charge.c/h         # 充电管理
│   ├── discharge.c/h      # 放电管理
│   └── boost_control.c/h  # 升压控制
├── output/
│   ├── usb_port.c/h       # USB-C端口管理
│   ├── usb_a_port.c/h     # USB-A端口管理
│   └── wireless.c/h       # 无线充电控制
├── ui/
│   ├── led.c/h            # LED指示
│   ├── display.c/h        # LCD/数码管（如有）
│   └── button.c/h         # 按键处理
└── utils/
    ├── delay.c/h
    ├── soft_i2c.c/h
    └── flash.c/h          # 参数存储（EEPROM/Flash）
```

### 主循环逻辑

```c
int main(void) {
    System_Init();        // 时钟、外设初始化
    BMS_Init();           // 电池保护初始化
    PD_Init();            // 协议栈初始化
    Wireless_Init();      // 无线充初始化

    while (1) {
        BMS_Task();       // 监测电芯状态
        PD_Task();        // 处理PD协商
        Charge_Task();    // 充电管理
        Discharge_Task(); // 放电管理
        Wireless_Task();  // 无线充电轮询
        UI_Task();        // LED/显示更新
    }
}
```

### 关键设计点

| 模块 | 核心问题 |
|------|---------|
| PD协议栈 | 状态机实现（Source/Sink切换、功率协商） |
| BMS | SOC精确计算（电流积分+温度补偿） |
| 多口功率分配 | 动态调度，防止过载 |
| 保护响应 | 中断驱动，毫秒级响应 |

## 六、开发工具

### 中基国威 WRT-8 烧录器
- 用于芯片程序烧录
- 支持脱机烧录（量产用）
- 需确认芯片型号以确定具体支持情况

### 芯片选型建议
中基国威方案（GC8xxx系列）在移动电源领域较为常见，需确认：
- 是否支持双向65W PD快充
- 是否支持多口100W输出
- 是否支持无线充电Qi2 15W（需外加模块）
- 是否支持LiFePO4电池

## 七、开发路径

```
1. 方案选型
   确认芯片型号 → 对照规格书确认是否满足需求

2. 硬件设计
   参考官方参考设计（原理图/PCB）→ 修改外围电路

3. 固件开发
   厂商提供的基础库 → 添加私有逻辑

4. 调试
   协议一致性测试、功率校准、保护参数调优
```
