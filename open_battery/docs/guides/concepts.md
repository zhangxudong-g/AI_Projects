# 技术概念参考

本文档记录项目中用到的关键技术概念，聚焦在本项目中的实际用法。

---

## IIC 通信协议 (Inter-Integrated Circuit)

两线制同步串行总线，用于芯片间短距离通信（板内通信）。

- **SDA** — 数据线（双向），**SCL** — 时钟线（主机控制节奏）
- **同步** — 有时钟线，不需要双方约定波特率
- **半双工** — 同一时刻只能收或发
- **寻址** — 每个设备有地址（7位或10位），支持一主多从
- **速度** — 标准模式 100kHz，快速模式 400kHz

### 通信过程

```
主机发 Start  →  发设备地址(7bit) + 读/写位(1bit)  →  等待 ACK  →  收发数据  →  发 Stop
                   │                                              │
                   └─ 7位地址最多挂 128 个设备，每个设备认自己的地址才回应 ACK
```

### 数据传输

SCL 低电平时 SDA 可以改变数据（准备数据），SCL 高电平时 SDA 必须稳定（读取数据）。每个 bit 都在 SCL 的上升沿被采样。

### 本项目用法

SH68F06 作为主机，通过 P10(SDA)/P07(SCL) 控制两个从设备：

| 设备 | 地址类型 | 地址 | 功能 |
|------|---------|------|------|
| SW6301 | 10位 | 0x14B | 升压输出 |
| SW6303 | 7位 | 0x23 | 充电管理 |

使用软件 GPIO 模拟 IIC（不依赖硬件 IIC 模块），时序延时通过 `NOP3()`/`NOP8()` 实现。通信时需确保 SDA/SCL 有 4.7kΩ 上拉电阻。

**本项目**：SH68F06 作为主机，通过 P10(SDA)/P07(SCL) 控制 SW6301(0x14B) 和 SW6303(0x23)。时序延时通过 `NOP3()`/`NOP8()` 实现。

详见 [development-guide.md](development-guide.md) 的 IIC 通信章节。

## UART 串口通信

异步全双工串行协议，点对点通信，只需 TX/RX/GND 三根线。

- **异步** — 无时钟线，双方约定相同波特率
- **全双工** — TX/RX 独立，可同时收发
- **数据帧**：起始位(1bit) + 数据(8bit, LSB先发) + [校验] + 停止位(1bit) = 10bit/帧

| 波特率 | 字节/秒 | 用途 |
|--------|---------|------|
| 9600 | ~960 | 低速调试 |
| 115200 | ~11520 | Arduino 模拟器 |

**本项目**：P02 配置为 UART0 TX，波特率约 29400（8MHz/(8×34+1)），启动时发送 "111" 作为标识。接收中断已注册但未实现处理逻辑。可用于调试输出。

## NTC 温度检测

负温度系数热敏电阻，阻值随温度升高而降低。用于电池过温保护，防止充电时起火。

电路为分压结构：`VDD → [NTC] → ADC采样点 → [固定电阻] → GND`，温度变化导致 NTC 阻值变化，ADC 读到不同电压值。

本项目 NTC 的 ADC 采样部分已被注释掉（main.c:161-172），`ntc_flay` 标志位在 int.c 中赋值。当温度异常时停止充电（P14=1），正常时允许充电（P14=0）。参见 `GET_NTC_VALUE()`（main.c:153）。

## 电源术语

| 标记 | 含义 | 本项目对应 |
|------|------|-----------|
| **VDD** | 数字电路供电电压 | SH68F06 的供电输入，随电池电压变化 |
| **VCC** | 集电极供电，常与 VDD 互换 | - |
| **VSS** | 数字地 (GND) | - |
| **VBAT** | 电池电压 | 电池组总电压 (2~6 节串联) |

在本项目中的电压路径：`电池组 → SW6303 充电管理 → VDD (SH68F06 供电)`。

为什么要读 VDD：ADC 的 NTC 温度计算依赖分压公式，需要知道 VDD 的实际值才能准确算出 NTC 电阻。参见 `GET_VDD_VALUE()`（main.c:106）和 `GET_NTC_VALUE()`（main.c:153）。

## ADC 模数转换

将模拟电压转换为数字值，MCU 才能处理。

- SH68F06 内置 12 位 ADC，结果存于 `ADC_DH`(高8位) + `ADC_DL`(低4位)
- 内部 2.5V 基准电压，支持 8 次平均滤波 (`ADC_RVDD_AVG8`)

**本项目使用了两个通道**：

| 通道 | 函数 | 用途 | 代码位置 |
|------|------|------|---------|
| VDD/4 | `GET_VDD_VALUE()` | 系统供电电压，供 NTC 计算 | main.c:106 |
| P16 | `GET_ADC_VALUE()` | 底座电压检测，判断充电器接入 | main.c:130 |

ADC 操作流程：选通道 → 等1ms → 触发(`ADC_SOFT_TG`) → 等完成(`ADC_WAIT`) → 读结果 → 清标志。

## GPIO 位操作

8051 内核的 GPIO 支持强大的位寻址能力，可直接操作单个引脚。

```c
// SH68F06 特有的寄存器配置方式
P06_IN_EN;     // 配置为输入
P06_PU_EN;     // 使能上拉电阻
P06_OUT_EN;    // 配置为输出

// 直接读写引脚
P14 = 0;       // 拉低
P14 = 1;       // 拉高

// 标准位操作
P1 |= 0x01;    // 置位
P1 &= ~0x01;   // 复位
```

**本项目**：IO 初始化集中在 `IO_INIT()`（main.c:27），配置了 IRQ 引脚(P06/P20/P13)为下降沿中断输入，LED(P17/P12/P11)和照明(P14)为输出。

## 中断机制

8051 的中断是硬件驱动的事件响应机制，中断发生时 CPU 暂停当前任务，跳转执行中断服务函数(ISR)。

### 本项目使用的中断源

| 中断 | 向量号 | 触发源 | ISR | 代码位置 |
|------|--------|--------|-----|---------|
| INT0 | 0 | 外部中断0 | 未使用 | - |
| T0 | 1 | 定时器0 | `INT1Interrupt` | int.c |
| INT1 | 2 | 外部中断1(P13按键) | 已使能 | main.c:218 |
| INT5 | 10 | UART0 接收 | 空实现 | main.c:682 |

### 定时器中断（核心）

T0 配置为 1ms 周期中断，是整个系统的时间基准：

```c
SYSCK_DIV4;        // 16MHz → 4MHz
TMR0_CY = 0x7C;   // 1/4µs × 4 × 8 × 125 = 1000µs = 1ms
TMR0_C0 = 0xC3;   // 定时器模式配置
```

ISR 中执行：时间计数累加、按键扫描、LED 显示刷新。主循环根据这些计数器做 100ms 周期任务。

## BMS (Battery Management System)

电池管理系统，监控和管理电池组，确保安全和寿命。核心功能：电压监测（防过充/过放）、温度监测（NTC 过温保护）、电流保护（过流/短路）、电芯均衡、电量计算。

本项目没有独立的 BMS 芯片，SW6303 和 SW6301 分担了部分 BMS 功能：SW6303 负责充电侧（限流、过温、满充截止），SW6301 负责放电侧（过流、欠压保护）。多节串联电池组通常会外加专用 BMS 保护板做电芯级均衡。

## 看门狗定时器

硬件定时器，必须在超时前"喂狗"（复位计数器），否则自动复位 MCU。防止程序死机或跑飞。

```c
WDT_C = 0x20;    // 使能看门狗 (main.c:305)
CLR_WDT;          // 喂狗 (main.c:312, 主循环中)
```

**本项目**：在主循环 `while(1)` 开头喂狗。如果主循环卡死，看门狗超时后自动复位系统。注意 `sys.c` 中也有一处喂狗（main.c:312），属于冗余喂狗。
