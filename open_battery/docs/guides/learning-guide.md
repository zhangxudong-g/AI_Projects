## 第一章 电学基础

### 1.1 什么是电？

想象水在水管里流动。电就像水，电子像水分子，从高电压处流向低电压处。

```
电压(V)    ──►    推动电子流动的力量，像水压
电流(I)    ──►    电子流动的速度，像水流速
电阻(R)    ──►    阻碍电子流动的东西，像水管阻力

欧姆定律：V = I × R
```

| 术语 | 符号 | 单位 | 日常类比 |
|------|------|------|----------|
| 电压 | V | 伏特(V) | 水压 |
| 电流 | I | 安培(A) | 水流速 |
| 电阻 | R | 欧姆(Ω) | 水管阻力 |

### 1.2 电路是什么？

电想要流动，必须有一条完整的路。这就是**电路**。

```
┌─────────────────────────────────────┐
│                                     │
│   ┌────┐         ┌────┐           │
│   │电池│────────►│ LED│           │
│   └────┘         └────┘           │
│     │              │              │
│     └──────────────┘              │
│         (形成环路)                 │
└─────────────────────────────────────┘
```

- **通路**：电能够流过去，灯就亮
- **断路**：电流不通，灯就灭

### 1.3 高电平与低电平

单片机用"电"来表示数字：

| 电平状态 | 电压范围 | 表示意义 |
|----------|----------|----------|
| 高电平 | 2.5V ~ 5V | 1 (有电) |
| 低电平 | 0V ~ 0.5V | 0 (没电) |

```c
// 单片机语言
LED_ON();   // 输出高电平，灯亮
LED_OFF();  // 输出低电平，灯灭
```

### 1.4 接地（GND）

"地"就是电的**最低点**，所有电压都是相对于"地"来说的。

```
┌───────────────┐
│  电池 3.7V    │──────┐
│               │      │   相对于"地"
└───────────────┘      │   这个电池提供
      │                │   3.7V电压
   ┌──▼──┐             │
   │ GND │◄────────────┘
   └─────┘
```

## 第二章 编程入门

### 2.1 什么是编程？

编程就是**用人类能理解的语言写指令，让计算机执行**。

| 人类语言 | 计算机语言 |
|----------|------------|
| "灯亮" | `LED_ON();` |
| "等一秒" | `DELAY(1000);` |
| "如果电压低，就报警" | `if(voltage < 3.0) ALARM();` |

### 2.2 C语言基础

C语言是单片机开发最常用的语言。

#### 2.2.1 变量 - 存储数据的小盒子

```c
// 声明一个整数变量，名字叫battery_voltage
int battery_voltage;

// 给它赋个值
battery_voltage = 4200;  // 单位是毫伏(mV)，即4.2V

// 也可以声明时就赋值
int percentage = 75;  // 电量75%
```

#### 2.2.2 函数 - 封装好的动作

```c
// 定义一个函数：让LED闪烁
void LED_Blink(void) {
    LED_ON();   // 灯亮
    DELAY(500); // 等待500毫秒
    LED_OFF();  // 灯灭
    DELAY(500); // 等待500毫秒
}

// 调用这个函数
LED_Blink();  // 执行一次闪烁
```

#### 2.2.3 条件判断 - 如果...就...

```c
int battery_voltage = 3500;  // 毫伏

if (battery_voltage < 3600) {
    // 如果电压低于3.6V
    LED_Flash();  // LED闪烁表示低电量
}
```

#### 2.2.4 循环 - 重复做一件事

```c
// while循环：条件为真时就一直执行
while (1) {        // 1永远为真，所以是无限循环
    LED_ON();
    DELAY(500);
    LED_OFF();
    DELAY(500);
}
// 这段代码会让LED永远闪烁下去
```

### 2.3 程序的本质

计算机只能理解0和1，但人类写的是C语言。中间需要一个**翻译**过程：

```
┌─────────────┐      编译        ┌─────────────┐
│  C语言代码   │ ────────────►   │  机器码     │
│  main.c     │                 │  (HEX文件)  │
└─────────────┘                 └─────────────┘
     人类能看懂                        单片机能看懂
```

- **编译器**：把C语言翻译成机器码的工具（本项目用Keil uVision）
- **烧录**：把机器码写入单片机芯片的过程

## 第三章 单片机基础

### 3.1 什么是单片机？

**单片机（MCU）= 一台集成在单个芯片里的微型计算机**

```
┌─────────────────────────────────────┐
│                                     │
│   ┌─────────────────────────────┐ │
│   │           CPU               │ │  中央处理器 - 执行计算
│   │      (大脑)                  │ │
│   └─────────────────────────────┘ │
│   ┌─────────────────────────────┐ │
│   │        存储器                │ │  存放程序和数据
│   │   (内存+Flash)               │ │
│   └─────────────────────────────┘ │
│   ┌─────────────────────────────┐ │
│   │      输入/输出引脚          │ │  连接外部设备
│   │        (GPIO)               │ │
│   └─────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘
              SH68F06芯片
```

### 3.2 本项目的单片机：SH68F06

| 参数 | 说明 |
|------|------|
| 内核 | 8051（经典、简单的CPU架构） |
| 位宽 | 8位（一次处理8个二进制位） |
| 主频 | 8MHz（内部振荡器） |
| 引脚数 | 20个 |
| 特点 | 强大的位操作能力，适合控制 |

### 3.3 单片机的"眼睛" - GPIO

GPIO = General Purpose Input Output（通用输入输出）

每个引脚可以配置为**输入**或**输出**：

```
输入模式（单片机读取外部信号）
┌────────┐    ┌────────────┐
│  按键  │───►│  P00引脚   │──► 单片机知道按键被按下
└────────┘    └────────────┘

输出模式（单片机控制外部设备）
┌────────┐    ┌────────────┐
│  LED   │◄───│  P17引脚   │◄── 单片机让LED亮
└────────┘    └────────────┘
```

### 3.4 什么是时钟？

时钟就像单片机的"心跳"，告诉CPU多快执行一条指令。

```
时钟 8MHz = 每秒震动8,000,000次
      = 每1/8微秒执行一条指令
```

## 第四章 固件与嵌入式系统

### 4.1 什么是固件？

| 软件类型 | 运行在哪 | 例子 |
|----------|----------|------|
| 应用软件 | 电脑/手机 | 微信、游戏 |
| **固件** | **单片机** | 充电宝程序、空调程序 |
| 驱动程序 | 操作系统 | 显卡驱动 |

**固件 = 写在单片机里、专门控制特定硬件的程序**

### 4.2 嵌入式系统有什么特点？

| 特点 | 含义 |
|------|------|
| 实时性 | 必须及时响应，如过流保护要在毫秒内反应 |
| 资源有限 | 没有硬盘，存储空间只有几KB |
| 无界面 | 不需要屏幕，通过LED或串口输出信息 |
| 稳定可靠 | 要长期稳定运行，不能死机 |

### 4.3 什么是"裸机"编程？

有些单片机不运行操作系统，程序直接控制硬件，这叫**裸机编程**。

```
有操作系统：应用软件 ──► 操作系统 ──► 硬件
裸机程序：  程序      ──►          硬件（更直接）
```

本项目就是裸机编程，没有操作系统，程序直接控制各芯片。

## 第五章 移动电源原理

### 5.1 电池基础

充电宝内部用的是**锂电池**：

| 类型 | 电压 | 特点 |
|------|------|------|
| 锂离子 | 3.7V | 常见于手机 |
| 磷酸铁锂 | 3.2V | 更安全、寿命长 |

**电池容量**：用mAh（毫安时）表示

```
5000mAh意味着：
如果用500mA电流放电，可以放10小时
如果用1000mA电流放电，可以放5小时
```

### 5.2 升压（Boost）原理

手机需要5V充电，但电池只有3.7V，怎么办？**升压**！

```
电池(3.7V)  ────►  升压芯片  ────►  USB输出(5V)
   │                        │
   │                  把电压升高
   ▼
消耗电池电能
```

升压芯片（SW6301）通过快速开关电路，把低电压"泵"成高电压。

### 5.3 降压（Buck）充电原理

充电时外部电压更高（5V/9V/12V），需要降到适合电池的电压：

```
充电器(5V~12V)  ────►  充电管理芯片  ────►  电池(4.2V限压)
```

充电管理芯片（SW6303）控制充电电流，充满自动停止。

### 5.4 移动电源工作模式

| 模式 | 条件 | 过程 |
|------|------|------|
| **放电模式** | 接手机/负载 | 电池 → SW6301升压 → USB输出5V |
| **充电模式** | 接充电器 | 充电器 → SW6303降压 → 电池 |
| **休眠模式** | 无操作/低电量 | 关闭大部分电路，省电 |
| **保护模式** | 异常情况 | 过温/过流/过放时停止工作 |

## 第六章 本项目芯片架构

### 6.1 三芯片协作

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   ┌─────────────┐           ┌─────────────┐   │
│   │   SW6301    │◄───IIC───►│   SH68F06   │   │
│   │  升压输出    │           │   主控MCU   │   │
│   │  0x14B地址  │           │   (大脑)    │   │
│   └─────────────┘           └──────┬──────┘   │
│                                    │           │
│                                    │IIC        │
│                           ┌────────▼───────┐   │
│                           │    SW6303      │   │
│                           │   充电管理     │   │
│                           │   0x23地址     │   │
│                           └────────────────┘   │
│                                                 │
└─────────────────────────────────────────────────┘
```

| 芯片 | 角色 | 主要职责 |
|------|------|----------|
| **SH68F06** | 主控（大脑） | 逻辑判断、状态机、按键处理、命令协调 |
| **SW6301** | 升压输出IC | 把电池电压升到5V/9V/12V输出 |
| **SW6303** | 充电管理IC | 控制输入充电电流、充满检测 |

### 6.2 IIC地址

每台设备在IIC总线上都有自己的"门牌号"：

| 芯片 | 地址类型 | 地址值 |
|------|----------|--------|
| SW6301 | 10位地址 | 0x14B |
| SW6303 | 7位地址 | 0x23 |

```
为什么地址不一样？
- SW6301使用10位地址，需要发送两次（先高两位，再低8位）
- SW6303使用标准7位地址，一次发送即可
```

### 6.3 引脚分配

| 引脚 | 功能 | 说明 |
|------|------|------|
| P10 | SDA | IIC数据线 |
| P07 | SCL | IIC时钟线 |
| P17~P14 | LED1~LED4 | 电量指示灯 |
| P11 | DC_EN | 升压输出使能 |
| P00 | KEY | 电源按键 |
| P21 | DET | 负载/充电器检测 |

## 第七章 IIC通信协议

### 7.1 什么是通信协议？

两个人对话需要共同语言。芯片对话也需要共同规则，这就是**协议**。

IIC协议的关键要素：

| 要素 | 说明 |
|------|------|
| 两根线 | SDA(数据) + SCL(时钟) |
| 主从结构 | 主机(SH68F06)发起命令，从机响应 |
| 同步通信 | 用时钟线同步，双方不需要约定速度 |

### 7.2 IIC通信过程

**场景：主机读取SW6303的充电状态**

```
主机                    SW6303
 │                        │
 │──── Start信号 ────────►│  (通知总线要开始了)
 │                        │
 │──── 发送地址+写(0x23) ─►│  (问SW6303:你在吗?)
 │                        │
 │◄─── ACK应答 ───────────│  (从机回答:在!)
 │                        │
 │──── 发送寄存器地址0x22─►│  (问:0x22寄存器的值是多少?)
 │                        │
 │──── Restart信号 ───────►│  (通知要读数据了)
 │                        │
 │──── 发送地址+读(0x23) ─►│  (再次确认是SW6303)
 │                        │
 │◄─── 数据0x01 ──────────│  (回答:0x01表示正在充电)
 │                        │
 │──── Stop信号 ──────────►│  (通信结束)
 │                        │
```

### 7.3 开始/停止信号

| 信号 | SDA状态 | SCL状态 | 含义 |
|------|---------|---------|------|
| Start | 高→低 | 高 | "我要开始说话了" |
| Stop | 低→高 | 高 | "我说完了" |

```
Start:  SCL高时，SDA从高变低
Stop:   SCL高时，SDA从低变高

   SCL ─┐  ┌──┐  ┌──┐
        └──┘  └──┘
   SDA  ─┐     ┌──┐
     └──┘     └──┘
     Start    Stop
```

### 7.4 应答信号（ACK）

每发送一个字节后，接受方必须回应：

```
发送方           接收方
   │                │
   │─── 8位数据 ───►│
   │                │
   │◄─── ACK位 ─────│  (低电平=收到，高电平=没收到)
   │                │
```

### 7.5 软件模拟IIC

本项目用GPIO引脚**手动制造**IIC时序，而不是用硬件IIC模块：

```c
// 模拟IIC的Start信号
void IIC_Start(void) {
    SDA = 1;  // 数据线先拉高
    SCL = 1;  // 时钟线拉高
    NOP8();   // 短暂延时
    SDA = 0;  // 数据线变低（Start条件）
    NOP8();
    SCL = 0;  // 时钟线变低，准备发送数据
}

// 发送一个字节
void SendByte(unsigned char dat) {
    for(i=0; i<8; i++) {
        SDA = dat & 0x80;  // 发送最高位
        SCL = 1;           // 通知对方读数据
        NOP8();
        SCL = 0;
        dat <<= 1;         // 左移，准备下一位
    }
}
```

## 第八章 寄存器入门

### 8.1 什么是寄存器？

**寄存器 = 芯片里的小箱子**，每个箱子有一个地址（门牌号），里面存放特定含义的值。

```
SW6301芯片内部
┌─────────────────────────────┐
│  地址0x00  │  保留          │
│  地址0x01  │  保留          │
│  地址0x02  │ OUTPUT_VOL     │ ◄── 存放输出电压值
│  地址0x03  │ OUTPUT_CUR     │ ◄── 存放输出电流值
│  地址0x04  │ STATUS         │ ◄── 存放工作状态
│  ...       │  ...           │
└─────────────────────────────┘
```

### 8.2 为什么需要寄存器？

芯片不知道你想要什么，通过寄存器来**配置**和**查询**：

| 操作 | 目的 | 例子 |
|------|------|------|
| **写寄存器** | 配置芯片行为 | "把输出电压设为5V" |
| **读寄存器** | 获取芯片状态 | "现在输出电流是多少？" |

### 8.3 SW6301寄存器

| 地址 | 名称 | 说明 | 常用值 |
|------|------|------|--------|
| 0x02 | OUTPUT_VOL | 输出电压 | 0x4B=5V, 0x6B=9V, 0x7B=12V |
| 0x03 | OUTPUT_CUR | 输出电流限制 | 值×100mA |
| 0x04 | STATUS | 工作状态 | 各状态位 |

### 8.4 SW6303寄存器

| 地址 | 名称 | 说明 | 常用值 |
|------|------|------|--------|
| 0x20 | INPUT_VOL | 输入电压检测 | bit0=1表示12.6V |
| 0x21 | CHG_CUR | 充电电流 | - |
| 0x22 | STATUS | 充电状态 | 0x01=充电中, 0x02=充满 |

### 8.5 寄存器操作示例

```c
// 设置SW6301输出5V
R_Address = 0x02;      // 选择OUTPUT_VOL寄存器
R_Value = 0x4B;        // 0x4B对应5V
IIC_WriteByte();       // 通过IIC发送给SW6301

// 读取SW6303的充电状态
R_Address = 0x22;      // 选择STATUS寄存器
IIC_ReadByte();        // 读取数据到R_Value
if(R_Value & 0x01) {   // bit0=1表示正在充电
    // 充电中...
}
```

## 第九章 主循环与中断

### 9.1 什么是主循环？

固件程序永远不会"结束"，而是在一个**无限循环**里反复执行：

```c
int main(void) {
    // ========== 初始化 ==========
    System_Init();     // 配置时钟
    GPIO_Init();       // 配置引脚
    ADC_Init();        // 配置ADC
    IIC_Init();        // 配置IIC
    Timer_Init();      // 配置定时器

    // ========== 主循环 ==========
    while(1) {         // 永远为真，死循环
        CLR_WDT();     // 喂狗，防止重启

        if(power_time_100ms) {    // 每100ms执行一次
            Check_Charger();       // 检查充电器
            Check_Load();          // 检查负载
        }

        if(key_time_10ms) {       // 每10ms执行一次
            Scan_Key();            // 扫描按键
        }

        LED_Update();             // 更新LED显示
    }
}
```

### 9.2 为什么需要中断？

有些事情**必须立即处理**，不能等主循环慢慢来：

| 情况 | 如果等主循环会怎样 | 中断如何解决 |
|------|-------------------|--------------|
| 按键按下 | 可能漏检 | 按键一按下就立即处理 |
| 定时到了 | 时间不精确 | 精确的1ms中断 |
| 过流发生 | 可能损坏硬件 | 毫秒级保护响应 |

### 9.3 中断原理

```
正常执行主循环时：

    while(1) {
        Check_Load();    ◄─── 正在执行这里
        Scan_Key();
    }

突然定时器中断发生了：

    ────── 中断信号来了！──────►

    暂停主循环，跳转到中断处理函数：

    void Timer0_Isr(void) {
        time_counter++;    ◄─── 立即执行这个
    }

    完成后回到主循环继续执行：

        Check_Load();    ◄─── 从这里恢复
```

### 9.4 本项目的中断使用

| 中断源 | 周期 | 作用 |
|--------|------|------|
| T0定时器 | 1ms | 时间基准、按键扫描、LED刷新 |
| INT1 | 下降沿 | 按键按下事件 |

### 9.5 定时器初始化

```c
void Timer_Init(void) {
    SYSCK_DIV4;           // 系统时钟4分频：16MHz→4MHz
    TMR0_CY = 0x7C;       // 定时周期配置
    TMR0_C0 = 0xC3;       // 定时器模式配置
    // 4MHz / 4 / 8 / 125 = 1000Hz = 1ms周期
}
```

### 10.4 关键代码解读

#### 10.4.1 GPIO初始化

```c
void GPIO_Init(void) {
    // LED引脚配置为输出
    P17_OUT_EN;   // LED1
    P12_OUT_EN;   // LED2
    P11_OUT_EN;   // LED3/升压使能
    P14_OUT_EN;   // LED4

    // 按键引脚配置为输入
    P06_IN_EN;    // 按键输入
    P06_PU_EN;    // 使能上拉

    // 初始化电平
    LED1 = 0;     // 灯灭
    LED2 = 0;
    LED3 = 0;
    LED4 = 0;
}
```

#### 10.4.2 IIC读取数据

```c
void IIC_ReadByte(void) {
    IIC_Start();              // 开始信号
    SendByte(SW6301_ADDR_W);  // 发送地址+写
    SendByte(R_Address);      // 发送寄存器地址
    IIC_Start();              // 重新开始
    SendByte(SW6301_ADDR_R); // 发送地址+读
    R_Value = RcvByte();      // 接收数据
    IIC_Stop();               // 停止信号
}
```

#### 10.4.3 状态检测

```c
// 检测充电器是否插入
void Check_Charger(void) {
    if(P21 == 0) {  // P21低电平表示充电器插入
        // 充电器插入，进行充电管理
        ADC_Sample();        // 采样电池电压
        Check_NTC();         // 检查温度
        Set_ChargeCurrent(); // 设置充电电流
    } else {
        // 充电器移除
        Disable_Charge();
    }
}
```
