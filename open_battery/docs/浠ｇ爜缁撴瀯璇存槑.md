# 代码结构说明

## 概述

本说明文档旨在区分 mini-Lite 移动电源项目中 **自己编写的代码** 和 **厂商提供的代码**，帮助开发者在修改和维护代码时快速定位。

---

## 项目目录结构

```
AI 电源/
├── arduino_iic_simulator/      # Arduino IIC 模拟器（自己写）
├── demo/
│   ├── TMR1_111/                # 主固件项目
│   │   ├── src/                 # 自己写的核心代码
│   │   ├── prj/                 # 自己写的项目代码
│   │   └── vendor/              # 厂商提供的代码
│   └── user_com/                # 厂商提供的通信库
├── docs/                        # 项目文档
└── CLAUDE.md                    # 项目说明
```

## 自己写的代码

### 1. Arduino IIC 模拟器

**目录**: `arduino_iic_simulator/`

| 文件 | 说明 | 功能 |
|------|------|------|
| `arduino_iic_simulator.ino` | Arduino 主程序 | 模拟 SW6301/SW6303 芯片 |

**特点**：
- ✅ 完全自主开发
- ✅ 用于无硬件时的开发和调试
- ✅ 提供 IIC 通信监控和模拟功能
- ✅ 支持串口命令控制

### 2. 主固件核心代码

**目录**: `demo/TMR1_111/src/`

| 文件 | 说明 | 功能 |
|------|------|------|
| `main.c` | 主程序 | 系统初始化、主循环、IO配置 |
| `main.h` | 主程序头文件 | 函数声明、包含文件 |
| `int.c` | 中断处理 | INT0、INT1 中断服务函数 |

**特点**：

**目录**: `demo/TMR1_111/prj/`

## 修改建议

### 自己写的代码（可以安全修改）

#### 优先级 1 - 高频修改
- `main.c` - 主逻辑和初始化
- `KEY.c/h` - 按键功能调整
- `LED.c/h` - 显示效果定制
- `sys.c/h` - 系统控制逻辑

#### 优先级 2 - 中频修改
- `int.c` - 中断处理优化
- `sleep.c` - 功耗管理调整
- `IIC.c/h` - 通信封装优化

#### 优先级 3 - 低频修改
- `main.h` - 头文件声明
- 头文件中的宏定义

---

### 厂商提供的代码（谨慎修改）

#### 不建议修改
- `STARTUP_F06.A51` - 启动代码，风险极高
- `SW6301.c/h` - 芯片驱动，可能影响硬件
- `SW6303.c/h` - 芯片驱动，可能影响硬件

#### 可谨慎修改
- `config.c/h` - 配置参数，需要了解含义
- `delay.c` - 延时函数，影响时序
- `iic_*.c/h` - IIC 驱动，影响通信时序
- `user_com.c/h` - 通信库，影响通信功能

## 开发工作流

### 1. 新增功能

```
1. 在 prj/ 目录创建新模块（如 NEWFUNC.c/h）
2. 在 main.c 中包含头文件
3. 调用新模块的函数
4. 编译测试
```

### 2. 修改现有功能

```
1. 定位到对应文件（如 KEY.c）
2. 理解现有代码逻辑
3. 修改代码
4. 编译测试
5. 使用 Arduino 模拟器验证
```

### 3. 调试问题

```
1. 使用 Arduino 模拟器观察 IIC 通信
2. 检查自己写的代码逻辑
3. 如果确认是自己代码的问题，直接修改
4. 如果怀疑是厂商代码问题：
   - 先检查配置参数
   - 查看数据手册
   - 最后才考虑修改厂商代码
```

## 文件命名规范

- 文件名：全大写或驼峰命名（如 `KEY.c`, `sys.c`）

## 版本控制建议

### Git 仓库结构（建议）

```
git init
.gitignore
  ├── *.hex          # 固件文件不提交
  ├── *.obj          # 编译中间文件不提交
  ├── *.lst          # 列表文件不提交
  ├── *.uvopt        # Keil 配置文件不提交
  ├── Output/        # 输出目录不提交
  └── .serena/       # 分析工具文件不提交
```

### 提交策略
- 自己写的代码：频繁提交，每次功能完成后提交
- 厂商提供的代码：初始提交后，除非有明确的修改需求，否则不提交

## 总结

### 关键要点

1. **自己写的代码** 位于：
   - `arduino_iic_simulator/` - Arduino 模拟器
   - `demo/TMR1_111/src/` - 核心代码
   - `demo/TMR1_111/prj/` - 功能模块

2. **厂商提供的代码** 位于：
   - `demo/TMR1_111/vendor/` - 芯片驱动
   - `demo/user_com/` - 通信库

3. **修改原则**：
   - 优先修改自己写的代码
   - 谨慎修改厂商代码
   - 修改前务必备份

4. **调试工具**：
   - Arduino 模拟器 - 无硬件调试
   - Keil uVision - 仿真器调试
   - 串口监控 - 观察通信
