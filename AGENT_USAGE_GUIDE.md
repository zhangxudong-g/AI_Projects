# AI Project Agents 使用指南

本项目包含多个AI代理，用于自动化各种开发任务。以下是每个代理的使用方法：

## 1. Git Auto Commit Agent
- **用途**: 自动提交和推送代码更改
- **触发词**: "提交", "收工", "功能完成", 或类似完成工作的表达
- **功能**: 自动添加更改、提交并推送到远程仓库

## 2. Code Review Agent
- **用途**: 代码审查和质量检查
- **触发方式**: 明确请求代码审查时使用
- **功能**: 
  - 检查代码风格和最佳实践
  - 识别潜在bug和安全漏洞
  - 提供建议改进措施

## 3. Testing Agent
- **用途**: 运行测试和分析测试结果
- **触发方式**: 请求运行测试或检查测试覆盖时使用
- **功能**:
  - 执行单元和集成测试
  - 生成测试报告
  - 建议增加测试覆盖

## 4. Documentation Agent
- **用途**: 文档创建和维护
- **触发方式**: 需要创建或更新文档时使用
- **功能**:
  - 生成API文档
  - 更新README文件
  - 维护技术文档

## 5. Bug Detection Agent
- **用途**: 检测和分析代码中的bug
- **触发方式**: 需要识别潜在问题时使用
- **功能**:
  - 识别潜在bug
  - 按严重程度分类
  - 提供修复建议

## 6. Project Init Agent
- **用途**: 项目初始化和环境设置
- **触发词**: "初始化", "init", "setup"
- **功能**:
  - 安装项目依赖
  - 创建必要的环境文件和目录
  - 设置虚拟环境
  - 配置环境变量
  - 注意：仅安装依赖和创建文件，不会启动项目

## 7. Performance Monitoring Agent
- **用途**: 监控应用程序性能和资源使用
- **触发方式**: 需要检查性能指标或优化系统时使用
- **功能**:
  - 收集CPU、内存、磁盘使用率等系统指标
  - 监控应用程序响应时间和吞吐量
  - 识别性能瓶颈和资源密集型操作
  - 生成性能报告和趋势分析
  - 提供性能优化建议

## 8. Dependency Management Agent
- **用途**: 管理项目依赖和安全检查
- **触发方式**: 需要更新依赖或检查安全问题时使用
- **功能**:
  - 扫描项目依赖以识别过时的包
  - 检查依赖中的安全漏洞
  - 建议依赖版本更新
  - 分析依赖关系图以识别冲突
  - 生成依赖健康报告

## 9. Autonomous Thinking Agent
- **用途**: 自主分析任务并决定使用哪些agents
- **触发方式**: 当需要智能决策使用哪些工具时使用
- **功能**:
  - 分析用户请求以理解任务需求
  - 确定哪些agents组合最适合完成任务
  - 自动按正确顺序触发必要的agents
  - 在多个agents之间进行协调
  - 监控执行情况并提供综合反馈
  - 根据上下文决定何时使用哪个agents

## 如何使用

要使用这些agents，请在与AI交互时明确提及您想要使用的agent类型，例如：
- "请进行代码审查" → 触发Code Review Agent
- "运行测试" → 触发Testing Agent
- "更新文档" → 触发Documentation Agent
- "检测潜在bug" → 触发Bug Detection Agent
- "提交更改" → 触发Git Auto Commit Agent
- "初始化项目" → 触发Project Init Agent
- "检查性能" → 触发Performance Monitoring Agent
- "检查依赖" → 触发Dependency Management Agent
- "智能分析任务" → 触发Autonomous Thinking Agent

AI将自动选择适当的agent来处理您的请求。
