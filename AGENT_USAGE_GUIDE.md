# AI Project Agents 使用指南

本项目包含多个AI代理，用于自动化各种开发任务。以下是每个代理的使用方法：

## 1. Git Auto Commit Agent
- **用途**: 自动提交和推送代码更改
- **触发词**: "提交", "收工", "功能完成", 或类似完成工作的表达
- **功能**: 自动添加更改、提交并推送到远程仓库

## 2. Code Review Agent
- **用途**: 代码审查和质量检查
- **触发方式**: 明确请求代码审查时使用
- **功能**: 
  - 检查代码风格和最佳实践
  - 识别潜在bug和安全漏洞
  - 提供建议改进措施

## 3. Testing Agent
- **用途**: 运行测试和分析测试结果
- **触发方式**: 请求运行测试或检查测试覆盖时使用
- **功能**:
  - 执行单元和集成测试
  - 生成测试报告
  - 建议增加测试覆盖

## 4. Documentation Agent
- **用途**: 文档创建和维护
- **触发方式**: 需要创建或更新文档时使用
- **功能**:
  - 生成API文档
  - 更新README文件
  - 维护技术文档

## 5. Bug Detection Agent
- **用途**: 检测和分析代码中的bug
- **触发方式**: 需要识别潜在问题时使用
- **功能**:
  - 识别潜在bug
  - 按严重程度分类
  - 提供修复建议

## 如何使用

要使用这些agents，请在与AI交互时明确提及您想要使用的agent类型，例如：
- "请进行代码审查"
- "运行测试"
- "更新文档"
- "检测潜在bug"
- "提交更改"

AI将自动选择适当的agent来处理您的请求。
